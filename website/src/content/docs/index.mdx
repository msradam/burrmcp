The hero above is a real run: a 1T-parameter open model (Kimi K2.6) investigating
a live incident on rails. Each query is recorded as evidence, out-of-phase calls
are refused, and the conclusion stays gated until the evidence cross-references.

## A state machine you can *see*, and a log you can *read*

An incident-response workflow: four states, the allowed transitions between them,
and one refusal. The agent attempted `close_incident` from `engaged`; that action
is not reachable there, so Theodosia refused it, recorded the attempt, and did not
advance the state.

## Why this shape works

LLM agents fail at procedural work in structural ways: they skip steps, stop too
early, or declare success without verifying. Research on why multi-agent systems
fail ([MAST](https://arxiv.org/abs/2503.13657), Cemri et al.) finds the
interventions that held came from architecture, external verification and a
termination-enforcing state machine, not prompt tweaks.
Theodosia is that state machine, on the wire. The rails fix the structure without
limiting the model: it keeps its full toolset and chooses freely within each step.
See it concretely in the [case study](/theodosia/case-study/): the same model on
the same incident tasks, run free-ranging versus on rails, with the benchmark's
own grader as the witness.
